# PRD: GasRisk AI

## 1. 제품 개요

### 1.1 제품명

**GasRisk AI**  
지역별 가스 사고 위험 예측 및 AI 신고 지원 Web 서비스

### 1.2 한 줄 설명

**GasRisk AI는 한국가스안전공사 공공데이터와 기상·건축물·지역 데이터를 결합하여 지역별 가스 사고 위험을 시계열로 예측하고, 시민이 사진과 신고 내용을 입력하면 AI가 위험 의심 요소와 신고 가이드를 제공하는 웹 서비스이다.**

### 1.3 서비스 형태

| 구분 | 내용 |
|---|---|
| 서비스 유형 | Web 기반 AI 서비스 |
| 제출 부문 | 제품·서비스 개발 |
| 핵심 기술 | 시계열 예측, LLM, 멀티모달 AI, 위험지도, 신고 자동화 |
| 주요 사용자 | 시민, 소상공인, 지자체 안전 담당자, 가스 안전관리자 |
| 핵심 가치 | 사고 발생 전 위험 감지, 신고 품질 향상, 점검 우선순위 도출 |

## 2. 공모전 적합성

### 2.1 공모전 요구사항 대응

| 공모전 요구사항 | GasRisk AI 대응 |
|---|---|
| 산업통상부 및 산하 공공기관 데이터 활용 | 한국가스안전공사, 한국가스공사, 한국가스기술공사 데이터 활용 |

### 2.2 핵심 제출 포인트

- **창의성**: 시계열 예측 + 멀티모달 사진 분석 + LLM 신고 지원을 결합한 예방형 안전 서비스
- **구체성**: 공공데이터 기반 위험 점수 산식, Web 화면, API, 아키텍처 제시

- **문제해결 가능성**: 지역별 사고 위험을 사전에 감지하고 점검 우선순위 도출
- **AI 활용 확산성**: OpenAI API, Vision, TimesFM, LightGBM 기반 확장 가능한 AI 서비스 구조

## 3. 문제 정의

### 3.1 현재 문제

가스 사고는 한 번 발생하면 인명·재산 피해가 크지만, 일반 시민과 소상공인은 사고 위험을 사전에 인지하기 어렵다. 지자체와 안전관리 기관 역시 한정된 점검 인력으로 모든 시설을 동일하게 관리하기 어렵다.

### 3.2 Pain Point

| 사용자 | 문제 |
|---|---|
| 시민 | 가스 냄새, 배관 노후, 계량기 이상을 봐도 위험한지 판단하기 어려움 |
| 소상공인 | 음식점·숙박업 등 가스 사용 시설의 자가점검 역량 부족 |
| 지자체 | 지역별 사고 위험 변화와 점검 우선순위 판단 어려움 |
| 안전관리자 | 사고·신고·시설·기상 데이터를 통합적으로 보기 어려움 |
| 공공기관 | 사고 예방 중심의 데이터 기반 서비스 확산 필요 |

## 5. 핵심 사용자

## 5.1 시민 사용자

### 목적

집 주변, 상가, 주거지에서 가스 냄새나 시설 이상을 발견했을 때 위험 여부를 빠르게 확인하고 신고 행동을 안내받는다.

### 주요 사용 시나리오

1. 사용자가 웹에 접속한다.
2. 현재 위치 또는 지역을 선택한다.
3. 지역 가스 위험도를 확인한다.
4. 계량기나 배관 사진을 업로드한다.
5. AI가 위험 의심 요소를 설명한다.
6. 신고가 필요한 경우 신고서 초안을 생성한다.

## 5.2 소상공인 사용자

### 목적

음식점, 숙박업, 소규모 사업장에서 가스 시설을 자가점검하고 위험 신호를 조기에 확인한다.

### 주요 사용 시나리오

1. 사업장 주소를 입력한다.
2. 사업장 주변 위험도를 확인한다.
3. LPG 용기, 배관, 밸브 사진을 업로드한다.
4. AI가 점검 항목을 자동 체크한다.
5. 위험 의심 항목과 조치 가이드를 확인한다.

## 5.3 지자체·관리자 사용자

### 목적

지역별 사고 위험 변화와 신고 증가 지역을 확인하고, 점검 우선순위를 정한다.

### 주요 사용 시나리오

1. 관리자 대시보드에 접속한다.
2. 지역별 위험 등급 지도와 순위를 확인한다.
3. 전월 대비 위험 상승 지역을 확인한다.
4. AI가 생성한 위험 원인 리포트를 검토한다.
5. 점검 우선순위 목록을 다운로드한다.

# 6. 핵심 기능 요구사항

## 6.1 기능 1: 지역별 가스 사고 위험 예측 지도

### 설명

한국가스안전공사 사고·신고 데이터와 시설·기상·건축물 데이터를 결합하여 지역별 가스 사고 위험도를 예측하고 지도에 표시한다.

### 주요 화면 요소

| 요소 | 설명 |
|---|---|
| 지역 선택 | 시도/시군구 선택 |
| 위험 등급 | 낮음 / 주의 / 위험 / 고위험 |
| 예측 기간 | 이번 달 / 다음 달 |
| 위험 추세 그래프 | 최근 12개월 사고·신고 추세 |
| 위험 원인 요약 | LLM 기반 원인 설명 |

### 주요 출력 예시

```text
서울특별시 ○○구의 다음 달 가스 사고 위험도는 ‘위험’ 단계입니다.
최근 3개월 신고 접수 증가율이 높고, 노후 건축물 비율과 음식점 밀집도가 높은 지역입니다.
한파 예보가 있어 난방용 가스 사용 증가에 따른 주의가 필요합니다.
```

## 6.2 기능 2: 사진 기반 가스시설 자가점검

### 설명

사용자가 가스 계량기, 배관, 밸브, LPG 용기 사진을 업로드하면 OpenAI Vision 기반 멀티모달 AI가 위험 의심 요소를 분석한다.

### 입력

| 입력값 | 설명 |
|---|---|
| 시설 사진 | 계량기, 배관, 밸브, LPG 용기 등 |
| 위치 정보 | 선택 입력 |
| 시설 유형 | 사용자가 선택하거나 AI가 추정 |
| 추가 설명 | “가스 냄새가 난다”, “배관이 오래됐다” 등 |

### 출력

| 출력값 | 설명 |
|---|---|
| 시설 유형 | 계량기 / 배관 / 밸브 / LPG 용기 |
| 위험 의심 요소 | 부식, 꺾임, 파손, 환기 부족, 장애물 등 |
| 위험도 | 낮음 / 주의 / 위험 |
| 행동 가이드 | 환기, 화기 사용 금지, 전문기관 문의 등 |
| 신고서 초안 연결 | 신고 도우미 기능으로 이동 |

### 안전 문구

```text
본 결과는 AI가 사진에서 식별한 위험 의심 요소이며, 실제 안전 여부는 전문기관의 점검이 필요합니다.
긴급 상황이 의심되는 경우 즉시 119 또는 가스안전 관련 기관에 신고하세요.
```

## 6.3 기능 3: AI 신고 도우미

### 설명

사용자가 자연어로 신고 내용을 입력하면 LLM이 신고에 필요한 정보를 구조화하고, 신고서 초안을 생성한다.

### 입력 예시

```text
집 앞 계량기 근처에서 가스 냄새가 나고 배관이 오래돼 보여요. 어제부터 냄새가 심해졌어요.
```

### LLM 구조화 결과

| 항목 | 결과 |
|---|---|
| 신고 유형 | 가스 누출 의심 |
| 발생 위치 | 집 앞 계량기 근처 |
| 발생 시점 | 어제부터 |
| 위험 키워드 | 가스 냄새, 노후 배관 |
| 긴급도 | 높음 |
| 권장 행동 | 환기, 화기 사용 금지, 즉시 신고 |

### 신고서 초안

```text
가스 누출 의심 신고입니다.
발생 위치는 집 앞 계량기 주변이며, 어제부터 가스 냄새가 지속적으로 발생하고 있습니다.
배관이 노후되어 보이며, 누출 가능성이 의심됩니다.
현장 확인 및 안전 점검을 요청드립니다.
```

## 6.4 기능 4: 관리자 위험 리포트

### 설명

관리자 화면에서 지역별 위험 변화, 신고 증가 지역, 점검 우선순위를 AI 리포트로 제공한다.

### 리포트 구성

| 항목 | 설명 |
|---|---|
| 위험 상승 지역 TOP 10 | 전월 대비 위험도 상승 지역 |
| 주요 원인 | 신고 증가, 시설 밀도, 기상 요인, 노후건물 |
| 점검 우선순위 | 위험도 기반 점검 대상 지역 |
| 예상 사고 유형 | LPG, 도시가스, 고압가스 등 |
| 정책 제안 | 캠페인, 점검 강화, 취약지역 안내 |

# 8. 데이터 모델 설계

## 8.1 지역-월 단위 통합 테이블

| 컬럼명 | 설명 |
|---|---|
| `region_code` | 행정구역 코드 |
| `region_name` | 지역명 |
| `year_month` | 기준 월 |
| `gas_accident_count` | 가스 사고 건수 |
| `gas_report_count` | 신고 접수 건수 |
| `lpg_accident_count` | LPG 사고 건수 |
| `citygas_accident_count` | 도시가스 사고 건수 |
| `facility_count_total` | 가스시설 수 |
| `lpg_station_count` | LPG 충전소 수 |
| `specific_facility_count` | 도시가스 특정사용시설 수 |
| `old_building_ratio` | 노후 건축물 비율 |
| `restaurant_density` | 음식점 밀도 |
| `avg_temp` | 월평균 기온 |
| `min_temp` | 월최저 기온 |
| `rain_days` | 강수일수 |
| `heatwave_days` | 폭염일수 |
| `coldwave_days` | 한파일수 |
| `inspection_count` | 점검 건수 |
| `predicted_accident_count` | 예측 사고 건수 |
| `risk_score` | 최종 위험 점수 |
| `risk_level` | 위험 등급 |

## 8.2 위험 점수 산식 초안

```text
GasRiskScore(r,t) =
  w1 * PredictedAccidentCount(r,t)
+ w2 * ReportIncreaseRate(r,t)
+ w3 * FacilityDensity(r)
+ w4 * OldBuildingRatio(r)
+ w5 * WeatherRiskIndex(r,t)
+ w6 * RestaurantDensity(r)
- w7 * InspectionCoverage(r,t)
```

### 위험 등급

| 점수 구간 | 등급 |
|---:|---|
| 0~30 | 낮음 |
| 31~60 | 주의 |
| 61~80 | 위험 |
| 81~100 | 고위험 |

MVP에서는 가중치를 룰 기반으로 시작하고, 데이터가 확보되면 LightGBM 또는 XGBoost 기반으로 위험 점수를 보정한다.

# 9. AI/ML 요구사항

## 9.1 시계열 예측 모델

| 모델 | 역할 |
|---|---|
| Google TimesFM | 지역별 사고·신고 건수 추세 예측 |
| Prophet | 계절성 해석용 베이스라인 |
| LightGBM | 외생변수 기반 위험도 예측 |
| Isolation Forest | 신고 급증, 이상 패턴 탐지 |

### MVP 모델 구성

```text
1단계: Prophet 또는 TimesFM으로 월별 사고/신고 건수 예측
2단계: 시설·기상·노후건물 변수를 결합해 위험 점수 산출
3단계: LightGBM으로 위험 점수 보정
4단계: LLM이 예측 결과를 자연어 리포트로 변환
```

# 19. 데모 시나리오

## 19.1 시연 제목

**“가스 냄새가 나는 상황에서 시민과 지자체가 어떻게 AI 도움을 받을 수 있는가?”**

## 19.2 Step 1. 위험지도 확인

발표자가 특정 지역을 선택한다.

```text
서울특별시 ○○구
다음 달 가스 사고 위험도: 위험
주요 원인: 신고 증가, 노후건물 비율, 음식점 밀집도, 한파 영향
```

## 19.3 Step 2. 사진 점검

계량기 또는 LPG 용기 사진을 업로드한다.

```text
AI 분석 결과:
- 시설 유형: 가스 계량기
- 위험 의심 요소: 배관 연결부 노후 의심, 주변 장애물 존재
- 권장 행동: 환기, 화기 사용 금지, 전문기관 점검 권장
```

## 19.4 Step 3. 신고 도우미

사용자가 상황을 자연어로 입력한다.

```text
집 앞 계량기 근처에서 가스 냄새가 나고 배관이 오래돼 보여요.
```

AI가 신고서를 생성한다.

```text
가스 누출 의심 신고입니다.
집 앞 계량기 주변에서 가스 냄새가 발생하고 있으며, 배관 노후가 의심됩니다.
현장 확인 및 안전 점검을 요청드립니다.
```

관리자가 위험 상승 지역과 점검 우선순위를 확인한다.

이번 달 점검 우선지역:
1. ○○구
2. △△구
3. □□구

AI 분석:
최근 신고 증가율과 노후건물 비율이 높아 선제 점검이 필요합니다.

# 20. 최종 제품 포지셔닝

GasRisk AI는 단순한 사고 통계 조회 서비스가 아니다.

**공공데이터 기반 가스 사고 예측 모델**,  
**OpenAI Vision 기반 사진 자가점검**,  
**LLM 기반 신고 구조화 및 관리자 리포트**를 결합한  
**지역 안전 예방형 AI 서비스**다.

최종 메시지는 다음과 같다.

> GasRisk AI는 가스 사고를 사후 통계가 아닌 사전 예측의 대상으로 전환한다.  
> 지역별 사고·신고·시설·기상 데이터를 결합해 위험을 조기에 감지하고, 시민의 사진과 신고 내용을 AI가 구조화함으로써 지자체와 안전관리기관의 선제 대응을 지원한다.
