## 1. 배경 & 문제 정의

여름철 하천·호소의 유해남조류(녹조) 대량 증식은 상수원 안전과 친수활동(수영·낚시·수상레저)에 직접적인 위협이다. 국립환경과학원은 조류경보제를 통해 주요 지점의 수질·조류 자료를 측정·공개하고, 관(官) 차원의 발생 단계 예측도 제공한다. 다만 이 예측 로직·코드는 일반에 공개되어 있지 않아 재현·확장·검증이 어렵다.

**문제:** 공개 데이터만으로 재현 가능한 녹조 조기경보 모델이 오픈소스로 존재하지 않는다.

**차별화 포인트(대회 관점):**
- 역대 수상작 중 환경 분야는 소수로 경쟁 밀도가 낮다.
- "농도 값 예측"에 머무는 기존 접근과 달리, **임계 초과 조기경보**로 프레이밍한다.
- 관의 공식 예측을 대체하는 것이 아니라, **공개 코드·다지점 확장·피처 해석이 가능한 오픈소스 의사결정 지원 도구**를 만든다.

## 2. 목표 / 비목표

**목표(Goals)**
- 공개 데이터로 다음 측정 시점(≈ +7일)의 유해남조류 임계 초과 여부를 예측하는 트리 기반 모델 파이프라인 구축.
- 데이터 로딩 → EDA → 피처 엔지니어링 → 모델링 → 검증까지 **누수 없이 재현 가능한** 코드 제공.
- 예측 결과와 근거(피처 기여도)를 확인할 수 있는 간단한 리포트/대시보드.

**비목표(Non-Goals)**
- 관의 공식 조류경보 발령을 대체하거나, 법적 효력을 갖는 경보 제공.
- 실시간 스트리밍 인프라 구축(초기 범위 밖, §14 확장 단계에서 검토).
- 물리 기반 수질 시뮬레이션 모델 개발(데이터 기반 접근에 집중).

## 4. 성공 지표(Success Metrics)

### 4.1 모델 지표
임계 초과는 희소 이벤트(불균형)이므로 정확도(accuracy)는 지표로 쓰지 않는다.

| 지표 | 정의 / 목표 |
|---|---|
| **PR-AUC** | 불균형 상황의 주 지표. 지속(persistence) 베이스라인 대비 유의미한 향상 |
| **Recall @ 고정 Precision** | 놓치면 위험이 큰 문제 특성상 재현율 우선(예: Precision 0.5 지점에서의 Recall 보고) |
| **Lead time** | 임계 초과를 며칠 앞서 잡아내는가(다음 측정 시점 기준) |
| **지점 일반화** | 학습에 없던 지점/연도에서의 성능 저하 폭 |

### 4.2 프로젝트/대회 지표
- **재현성:** 클린 환경에서 README만 보고 end-to-end 재실행 성공.
- **문서화:** 각 모듈 docstring + 데이터 출처·라이선스 명시.
- **확장성:** 신규 지점 추가 시 코드 수정 없이 설정만으로 동작.

## 6. 타깃 정의(예측 대상)

**확정: 이진 분류 — 임계 초과 조기경보(안정화 후 다중분류로 확장).**
다음 측정 시점(≈ +7일)에 유해남조류 세포수가 임계 **1,000 cells/mL(관심 단계)**를 초과하는지 예측한다. (F2 근거: 양성률 11.6%, §16 Q4)

- **프레이밍(F2 확정):** 라벨은 **다음 측정치**로 생성하고(≈+7일), 예측 시점 t와 라벨 측정 시점 사이의 **경과일수를 통제**한다 — 경과일수를 피처/조건으로 두고, 측정 간격이 목표 시야(예: 4~10일) 밖인 샘플은 유효 샘플에서 제외한다. (F2: 91.3%가 5~8일 주간이라 손실 최소)
- 라벨 생성용 측정치를 **피처로 재사용하지 않는다.**

**대안(로드맵):**
- 다중분류(관심/경계/심각 단계): 정보량↑이나 상위 단계 희소로 초기엔 불안정 → 이진 안정화 후 확장.
- 회귀(세포수 값): 극단 분포·해석성 문제로 초기 범위 밖.

## 8. 모델링 전략

### 8.1 피처 엔지니어링
- **Lag:** 유해남조류·클로로필-a·수온 등의 과거값(1~N 측정 전).
- **Rolling:** 이동 평균·추세·변동성. **반드시 `shift(1)` 이후 rolling** 순으로 계산해 현재값 유입 차단.
- **계절:** 월/주차, 조류 시즌 여부, (가능하면) 누적 온도·일사.
- **기상:** 과거 기상 lag 또는 예보값만 사용(실측 미래 금지).
- **공간/지점:** `site`를 범주형으로. 초기엔 지점별 개별 학습과 통합 학습을 모두 실험.

### 8.2 모델 후보
- 1차(MVP): **LightGBM**(범주형·결측 처리 우수, 빠른 실험).
- 비교: XGBoost, CatBoost(범주형 자동 처리), 지속(persistence)·직전값 베이스라인.
- 불균형: class weight / scale_pos_weight, 임계값 튜닝(정밀-재현 트레이드오프).
- **딥러닝(확장 벤치마크):** 데이터가 작아 지점별로는 GBDT가 유리. 단 **전 지점을 묶은 글로벌 모델**로 만들면 표본이 커져 다계열 딥러닝(TFT·DeepAR·N-HiTS 등)이 후보가 됨. **교체가 아닌 비교(ablation)로 추가**하고 동일 시간순 CV에서 LightGBM을 이기면 승격(§16 Q6). GBDT와 정면 비교해 "이 규모에서 무엇이 더 적합한가"를 근거와 함께 제시(방법론 비교 자체가 산출물의 가치).

### 8.3 검증 전략 & 누수 방지 (프로젝트 최우선 원칙)
1. **미래 기상 실측값 금지** — t 시점에 알 수 있는 정보만 사용.
2. **lag/rolling 순서 준수 + 시간순 CV** — `groupby('site')` → 시간 정렬 → `shift` → rolling. 검증은 연도 홀드아웃 또는 `TimeSeriesSplit`, 무작위 KFold 금지.
3. **불규칙 샘플링 통제** — 측정 간격이 경보 수준에 의존하므로, 타깃은 경과일수를 통제해 생성하고 라벨 생성용 측정치를 피처로 재사용하지 않는다.
4. **지점 누수 주의** — 지점 통합 모델 평가 시, 미래 시점뿐 아니라 미지 지점 일반화도 별도 검증.
