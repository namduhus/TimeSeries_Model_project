# GasRisk AI 공식 활용 데이터 목록

본 문서는 GasRisk AI 서비스 구현을 위해 활용할 공공데이터 목록과 데이터별 활용 목적을 정리한 문서이다.  
공모전 요건에 따라 산업통상부 산하 공공기관인 한국가스안전공사 데이터를 핵심 데이터로 활용하며, 예측 성능과 서비스 설명력을 높이기 위해 기상청, 국토교통부, 행정안전부 등 타 기관 공공데이터를 보조적으로 연계한다.

# 4. 데이터 활용 관점별 매핑
> 수집된 데이터가 시스템 내부에서 어떻게 가공되고, 어떤 최종 산출물(Feature, 예측값, 서비스)로 연결되는지 정의합니다.

| 활용 목적 | 사용 데이터 | 산출물 |
| :--- | :--- | :--- |
| **시계열 사고 예측** | 가스사고 현황, 신고 접수 현황, 기상자료 | 다음 달 지역별 사고·신고 예측값 (TimesFM 활용) |
| **위험 노출도 산출** | 국내 가스시설 현황, LPG 충전소 현황, 도시가스특정사용시설 통계 | 시설 밀도, 고위험 시설 지수 (GIS 매핑 Feature) |
| **취약지역 보정** | 노후건물정보, 건축물대장, 인구·고령인구 통계 | 노후도 지수, 피해 취약도 지수 |
| **상권 위험 보정** | 음식점/숙박업 인허가 데이터, 소상공인 상권정보 | 가스 사용 업종 밀집도 |
| **관리자 리포트 생성** | 사고 예측값, 위험점수, 주요 Feature, 가스사고연감 | 위험 상승 원인 설명, 점검 우선순위 리포트 |
| **시민 신고 지원** | 가스사고예방사례집, 안전점검 체크리스트, 사용자 입력 텍스트·이미지 | AI 기반 신고서 초안, 행동요령, 자가점검 결과 |



# 5. 프로젝트 데이터 및 모델링 파이프라인

## 5.1 전체 파이프라인 개요

GasRisk AI의 모델링 파이프라인은 다음과 같이 구성한다.

1. **공공데이터 수집**

2. **지역-월 단위 데이터 통합**

3. **시계열 예측값 생성**

4. **위험점수 산출**

5. **LLM 기반 위험 원인 설명**

6. **멀티모달 기반 신고 지원**

7. **Web 서비스 제공**


## 5.2 Step 1. 데이터 수집

### 5.2.1 한국가스안전공사 핵심 데이터 수집
> GasRisk AI의 핵심 타깃(Target) 및 주요 Feature로 활용되는 한국가스안전공사의 기본 데이터셋입니다.

| 데이터셋 명 | 수집 및 활용 목적 |
| :--- | :--- |
| **가스사고 현황(월별 원인별)** | 지역·월별 사고 발생 추세 파악 및 TimesFM 예측 핵심 타깃 설정 |
| **가스사고 신고 접수 현황** | 사고 발생 전후의 신고 패턴 및 이상 급증 징후 탐지 |
| **국내 가스시설 현황** | 지역별 가스시설 밀도 및 기본 위험 노출도 산출 |
| **전국 LPG 충전소 현황** | LPG 충전소 공간(GIS) 위치 기반 고위험 영역 매핑 |
| **도시가스특정사용시설 통계** | 대용량 가스 사용 시설의 밀집도를 통한 위험 가중치 반영 |
| **가스사고연감** | LLM/RAG 기반 사고 유형별 상세 원인 설명 근거 확보 |
| **가스사고예방사례집** | 사용자가 사진/텍스트 입력 시 맞춤형 시민 행동요령 생성 |

<br>

### 5.2.2 외부 보조 데이터 수집
> 가스 사고 발생의 외부 요인을 분석하고 LightGBM 모델의 예측력을 높이기 위한 외생변수(External Feature) 데이터셋입니다.

| 데이터셋 명 | 제공 기관 | 수집 및 활용 목적 |
| :--- | :--- | :--- |
| **기상청 기상자료** | 기상청 | 기온, 강수, 폭염, 한파 등 계절성 및 환경 요인 반영 |
| **노후건물정보** | 행정안전부 | 30년 이상 노후 주거지 및 상가 밀집지역의 인프라 취약성 반영 |
| **건축물대장** | 국토교통부 | 건물 용도, 사용연수, 구조 정보를 활용한 시설 취약도 추정 |
| **음식점/숙박업 인허가 데이터** | 지방행정데이터 | 소상공인 및 가스 다소비 업종 밀집 상권 위험도 보정 |
| **인구·고령인구 통계** | 통계청 / 행안부 | 사고 발생 시 대피 취약층(고령층) 비율 기반 피해 취약도 보정 |


# 6. 서비스 제공 (Service Delivery)
* **시민용 Web:** 지역 위험도 조회, 사진 기반 자가점검, AI 신고서 초안 생성
* **지자체/관리자 Web Dashboard:** 지역별 GasRisk Score 모니터링, 위험 상승 지역 탐지, 점검 우선순위 관제
